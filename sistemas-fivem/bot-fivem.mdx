## BOT FIVEM

Visão geral

Funcionalidades são habilitadas dinamicamente com base no `config` do servidor. Cada servidor opera com um bot isolado e configurável via Dashboard ou JSON.

Arquitetura

- Per-server bot (instância por servidor)
- Configuração centralizada no Dashboard com fallback para arquivo JSON local
- Persistência em MySQL para whitelist, tickets e logs
- Permissões baseadas em cargos do Discord e roles mapeadas no config

Configuração

Campos de configuração (resumo)

Campo | Tipo | Descrição
:--|:--|:--
`sql.host` | string | Host do banco MySQL
`sql.user` | string | Usuário do banco
`sql.password` | string | Senha do banco (use env vars / painel)
`channels.welcome` | string | Canal de boas-vindas (ID)
`channels.exit` | string | Canal de logs de saída (ID)
`connect.join_url` | string | Link para entrar no servidor FiveM
`connect.server.vps` | string | IP:PORT da VPS do servidor
`automatic.auth_role` | string | Role atribuída após verificação
`whitelist.enabled` | boolean | Habilita fluxo de allowlist
`whitelist.questions` | array | Perguntas configuráveis para allowlist
`liberation.table` | string | Tabela no banco que será atualizada

Checklist de implantação (Boas práticas)

- Evitar chaves duplicadas no JSON (última sobrescreve).
- Não comitar tokens/secretos: use variáveis de ambiente ou secret store.
- Validar schema do config no startup (tipos, formatos de ID, URLs, limites).
- Manter versionamento das alterações de config no Dashboard (audit log).
- Fazer backup antes de aplicar mudanças em produção.

Segurança

- Tokens nunca em arquivos versionados.
- Permissões do Dashboard: limitar quem pode editar configs e reiniciar bots.
- Sanitização de inputs vindos do Dashboard para evitar injeção.

Lifecycle do bot

- Startup: validação do config → conexão ao MySQL → registrar handlers.
- Reload config: aceitar reload seguro sem reiniciar a instância quando possível.
- Restart por servidor: ação suportada no Dashboard com logs de mudança.

Observabilidade

- Logs estruturados para eventos: whitelist, tickets, liberações, erros.
- Métricas: conexões ativas, requisições de whitelist, tempo médio de aprovação.
- Auditoria: quem alterou o config e quando (Dashboard).
