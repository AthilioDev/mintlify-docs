# Athilio_Spotify

Athilio_Spotify é o resource standalone de música para servidores FiveM da Athilio Software. Ele oferece:

- painel Dashboard para controle de configurações
- validação de licença segura
- proteção por IP e bloqueio automático
- compatibilidade com bases customizadas e frameworks populares

<Info>
  Nesta documentação você encontra as etapas de instalação, configuração de licença, uso do painel e detalhes de compatibilidade por versão.
</Info>

## Primeira instalação

### Requisitos

- FiveM atualizado
- pasta `resources/` acessível no servidor
- `oxmysql` instalado quando aplicável
- licença ativa no painel Athilio

### Instalação do resource

1. Extraia `athilio_spotify` para a pasta `resources/`.
2. Confirme que a pasta está nomeada exatamente como `athilio_spotify`.
3. Adicione ao final do `server.cfg`:

```cfg
ensure athilio_spotify
```

4. Reinicie o servidor.

<Tip>
  Configure o `ensure athilio_spotify` após as dependências do servidor para evitar carregamento prematuro.
</Tip>

## Licença

O Athilio_Spotify usa autenticação de licença para validar o resource durante a inicialização.

### Arquivo de licença

A licença deve ser configurada em:

`athilio_spotify/license.json`

### Exemplo de arquivo

```json
{
  "license": "ATHILIO-XXXX-XXXX-XXXX"
}
```

### Como configurar

1. Acesse o painel da Athilio Software.
2. Copie sua licença oficial.
3. Abra `athilio_spotify/license.json`.
4. Cole a licença no campo `license`.
5. Salve o arquivo.
6. Reinicie o servidor.

<Warning>
  Nunca compartilhe sua licença publicamente. Compartilhamento, revenda ou uso em múltiplos servidores causa bloqueio automático.
</Warning>

### Verificação

Após a inicialização do servidor, verifique o console por esta mensagem:

`[Athilio Spotify] ✔ Licença validada com sucesso!`

Se essa mensagem aparecer, a licença foi autenticada corretamente.

## Configuração por versão

### Versão 1.0.2

Se você estiver usando a versão `1.0.2`, a configuração do programa é feita via `config.lua`.

```lua
Config = {}

Config.License = ""
Config.ServerName = "Core Studios Fy"
Config.HexColor = "27ae60"
Config.Logo = ""
Config.YoutubeAPIKey = ""
Config.Command = ""
Config.DefaultVolume = 50
Config.PropModel = ""
Config.PropFallback = ""
Config.EnableAnimation = true
Config.AnimDict = ""
Config.AnimName = ""
Config.PropBone = 57005
Config.PropPlacement = {0.26, -0.12, -0.10, 80.0, -160.0, 180.0}
```

<Note>
  Na versão 1.0.2, toda personalização do resource deve ser feita no arquivo `config.lua`. Faça backup antes de editar.
</Note>

### Versões acima de 1.0.2

Para versões mais recentes, todas as configurações são feitas diretamente pelo Dashboard da Athilio Software.

<Info>
  O Anti-Spoofer integrado está disponível nas versões acima de 1.0.2 e funciona junto com o painel de administração.
</Info>

## Dashboard

O Dashboard é o local central para:

- gerenciar licença e IP autorizado
- ajustar configurações do recurso
- atualizar informações do servidor
- visualizar histórico de validações

<Warning>
  A configuração do Dashboard só está disponível em versões recentes. Se a sua versão não mostrar a aba de configurações, use `config.lua` conforme a versão 1.0.2.
</Warning>

## Bloqueio por IP

O sistema valida o IP do servidor no momento da ativação.

- IP autorizado deve estar correto no painel.
- Migração de VPS ou mudança de IP exige redefinição.
- Uso em múltiplos servidores sem autorização aciona bloqueio.

<Warning>
  Licenças usadas em mais de um servidor são bloqueadas automaticamente. Sempre use uma licença por servidor.
</Warning>

## Problemas comuns

### Licença inválida

Verifique:
- licença copiada corretamente
- espaços extras no arquivo
- JSON formatado corretamente
- IP autorizado no painel

### Arquivo inválido

O arquivo deve permanecer exatamente no formato:

```json
{
  "license": "SUA_LICENSE_AQUI"
}
```

### Recurso não inicia

Verifique:
- `ensure athilio_spotify` no `server.cfg`
- nome da pasta `athilio_spotify`
- dependências carregadas antes do recurso
- cache do FiveM limpo

### Interface não abre

Verifique:
- pasta `html/` presente
- arquivos `index.html`, `style.css`, `script.js`
- erros no console `F8`
