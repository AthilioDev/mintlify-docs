## BOT FIVEM

Esta página descreve o bot FiveM usado por servidores integrados ao painel. O bot é instalado por servidor — cada servidor tem seu próprio bot exclusivo — e oferece todas as funcionalidades definidas no arquivo de configuração (SQL, canais, whitelist, tickets, liberação, etc.).

## Exemplo de configuração (sanitizado)

> Atenção: valores sensíveis (senhas, tokens) foram ocultados no exemplo abaixo. No ambiente real, armazene senhas em variáveis de ambiente ou no painel com criptografia.

```json
{
  "sql": {
    "host": "<DB_HOST>",
    "user": "<DB_USER>",
    "password": "<DB_PASSWORD_REDACTED>",
    "database": "<DB_NAME>",
    "port": 3306
  },

  "channels": {
    "welcome_channel": "<WELCOME_CHANNEL_ID>",
    "exit_channel": "<EXIT_CHANNEL_ID>"
  },

  "atendimento": {
    "category": "<ATENDIMENTO_CATEGORY_ID>",
    "staff": "<STAFF_ROLE_ID>"
  },

  "idservidor": {
    "id": "<SERVER_ID>",
    "image": "<SERVER_IMAGE_URL>"
  },

  "connect": {
    "link": "<FIVEM_JOIN_LINK>",
    "link2": "<STORE_URL>",
    "link3": "<INSTAGRAM_URL>",
    "link4": "<TIKTOK_URL>",
    "players": "128",
    "ipvps": "<VPS_IP:PORT>"
  },

  "automatico": {
    "authrole": "<AUTO_ROLE_ID>",
    "automessage": "<AUTO_MESSAGE_CHANNEL_ID>"
  }
}
```

O exemplo acima mostra os principais blocos. O bot também contém configurações completas de whitelist, tickets, categories de allowlist e módulo de liberação (rename/hex) — todos mapeados para canais, roles e tabelas do banco.

## Funções suportadas (mapeadas ao `config`)

- **Banco de dados (sql)**: Conexão MySQL para persistência (whitelist, tickets, registros).
- **Canais (channels)**: Canal de boas-vindas e log de saída para mensagens automáticas.
- **Atendimento (atendimento)**: Cria calls/threads de suporte em uma categoria controlada por cargo staff.
- **ID do servidor (idservidor)**: Identificador do servidor e imagem mostrada no painel.
- **Conexão FiveM (connect)**: Links de entrada, loja e redes sociais; número máximo de players e IP:PORT para conexão.
- **Automático (automatico)**: Role automática após verificação e canal de mensagens periódicas.
- **Whitelist**: Formulário configurável com imagem, limites, canais, cargos e perguntas (pleno suporte a perguntas longas e histórico).
- **Liberação (liberation)**: Fluxo para entregar/remover cargos e atualizar a tabela do jogo (ex.: `vrp_users`).
- **Ticket**: Sistema de tickets com categorias pré-configuradas e registro em tabela.

> Observação: o bot executa todas as funções declaradas no config — se um campo estiver ausente, a funcionalidade relacionada ficará desabilitada.

## Dashboard e edição

- **Edição via Dashboard**: Todos os campos essenciais podem ser alterados no painel — você pode editar valores manualmente (JSON) ou usar os botões/formulários disponíveis no Dashboard.
- **Ações**: Ligar, reiniciar e parar o bot por servidor com um clique. Mudanças simples (ex.: texto de canais) aplicam-se imediatamente; alterações de esquema (ex.: mudança de tabela) podem requerer reinício.
- **Per-server bot**: Cada servidor tem um bot exclusivo; configurações e limites são isolados por servidor.
- **Customização paga**: Podemos habilitar opções de customização adicionais (temas, plugins, limites maiores) por valores extras; essas opções aparecem no painel de customização.

## Limpeza e recomendações (LIMPE OS ARQUIVOS)

- **Remover chaves duplicadas**: No JSON exemplo que você forneceu havia chaves duplicadas (`allowlistcategories` apareceu duas vezes). JSON não aceita duas chaves iguais — a última sobrescreve as anteriores. Reúna todas as categorias em um único array.
- **Sanitizar segredos**: Nunca comite senhas/keys no repositório. Use variáveis de ambiente ou segredo do painel.
- **Validação**: Adicione validação do config no servidor (tipos, formatos de IDs, URLs, limites numéricos) para evitar comportamento inesperado.
- **Backups de config**: Quando o Dashboard editar a config, mantenha versões e permit a restauração.

## Exemplo rápido de fluxos

- Ao entrar: o bot envia a mensagem de boas-vindas no `welcome_channel` e inicia a verificação captcha.
- Verificação: usuário completa captcha → recebe `authrole` automaticamente.
- Whitelist: usuário preenche formulário; staff recebe notificação no canal configurado e aprova/reprova.
- Ticket: usuário abre ticket; um canal temporário é criado na categoria `ticket.category` e notifica cargos `ticket.staff`.
