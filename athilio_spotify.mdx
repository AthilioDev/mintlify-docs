# Athilio Spotify

`athilio_spotify` é o resource de música para FiveM que utiliza licença e painel centralizado para validação e configuração.

<Info>
  Este guia cobre instalação, licença, Dashboard Cloud e problemas de implantação.
</Info>

## Requisitos

- FiveM atualizado
- pasta `resources/` disponível no servidor
- `oxmysql` instalado quando necessário
- licença ativa no painel Athilio

## Instalação

1. Extraia `athilio_spotify` para `resources/`
2. Confirme que a pasta está nomeada `athilio_spotify`
3. Adicione no `server.cfg`:

```cfg
ensure athilio_spotify
```

4. Reinicie o servidor.

<Tip>
  Coloque `ensure athilio_spotify` após as dependências para evitar carregamento prematuro.
</Tip>

## Licença

A licença de uso deve ser informada em `athilio_spotify/license.json`.

### Exemplo

```json
{
  "license": "ATHILIO-XXXX-XXXX-XXXX"
}
```

### Passos

1. Acesse o painel Athilio.
2. Copie a licença ativa.
3. Abra `athilio_spotify/license.json`.
4. Cole a licença no campo `license`.
5. Salve o arquivo.
6. Reinicie o servidor.

<Warning>
  Não compartilhe sua licença publicamente. Uso indevido em vários servidores pode resultar em bloqueio.
</Warning>

### Verificação

No console, confirme a mensagem:

`[Athilio Spotify] ✔ Licença validada com sucesso!`

## Configuração por versão

### Versão 1.0.2

Nesta versão, a maioria das opções é configurada em `config.lua`.

```lua
Config = {}

Config.License = ""
Config.ServerName = "Core Studios Fy"
Config.HexColor = "27ae60"
Config.Logo = ""
Config.YoutubeAPIKey = ""
Config.Command = ""
Config.DefaultVolume = 50
Config.PropModel = ""
Config.PropFallback = ""
Config.EnableAnimation = true
Config.AnimDict = ""
Config.AnimName = ""
Config.PropBone = 57005
Config.PropPlacement = {0.26, -0.12, -0.10, 80.0, -160.0, 180.0}
```

<Note>
  Faça backup antes de editar `config.lua`.
</Note>

### Versões acima de 1.0.2

Para versões mais recentes, as configurações são feitas pelo Dashboard Cloud.

## Dashboard Cloud

O Dashboard Cloud é o painel central para gerenciar:

- licença e IP autorizado
- configurações do resource
- histórico de validações
- bloqueios e revogações

<Warning>
  Se sua versão não exibir o painel, use `config.lua` conforme a versão 1.0.2.
</Warning>

## Validação do resource

O resource valida a licença e o IP autorizado durante a inicialização.

- IP autorizado deve estar configurado no painel.
- Alteração de VPS ou IP exige atualização no painel.
- Uso em múltiplos servidores sem autorização causa bloqueio.

## Problemas comuns

### `license.json` inválido

Verifique:
- licença copiada corretamente
- formato JSON válido
- ausência de espaços extras
- campo `license` preenchido

### `athilio_spotify` não carrega

Verifique:
- `ensure athilio_spotify` no `server.cfg`
- nome da pasta `athilio_spotify`
- dependências carregadas antes do resource
- cache do FiveM limpo

### Dashboard não aparece

- Confirme a versão do resource
- Atualize para release mais recente
- Use `config.lua` apenas em `1.0.2`
